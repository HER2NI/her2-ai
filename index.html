What H.E.R does

The AI chat app that shows the live structure of your interaction.

Conversations with AI can drift, lose constraints, ignore format, forget context, or become messy over time. H.E.R maps the structure of the exchange and gives the system a cleaner interaction state to work from.

Preserves the thread

H.E.R tracks state, direction, and continuity so the conversation is less likely to collapse into repetition or drift.

Respects constraints

Receipts and audits help verify whether the response followed the user’s format, limits, and requested shape.

Works above models

Your relationship is with H.E.R. H.E.R can route through different AI providers while keeping the interaction layer coherent.

H.E.R-ID

Your portable interaction identity.

H.E.R-ID is a user-facing continuity signal. It is not intended to be exposed as a raw provider tracking handle or used as a raw memory owner key. It helps frame how you choose to work with H.E.R: your preferences, interaction patterns, response constraints, context boundaries, and consented memory.

H.E.R learns interaction patterns

Not raw private transcript by default. The value is how you interact: depth, pacing, format, correction style, and what helps you get better output.

Better over time

The longer you choose to use H.E.R through your H.E.R-ID, the more she can help shape the exchange around you.

Memory boundary

H.E.R lets you control what she learns.

The launch boundary is simple: memory is explicit, bounded, and visible. H.E.R should never need secrecy to be powerful.

Memory off

No durable learning. The system may operate for the current interaction, but no long-term H.E.R-ID memory is created.

Session memory

Chat-local. Useful for the current exchange without becoming portable identity memory.

Portable memory

Future-gated. Requires explicit consent, clear controls, export/delete paths, and a visible H.E.R-ID boundary.

Current iOS boundary: no backend database for portable H.E.R-ID memory; no cloud sync; provider payloads must not receive H.E.R-ID memory records. Provider names, if used, are descriptive compatibility context only; H.E.R is not affiliated with or endorsed by those providers unless expressly stated.
